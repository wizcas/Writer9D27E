# MVU 变量更新规则文件格式

> 本文件为人类可读的规则参考手册。CardMvuRules agent 提示词中应内嵌本规则的副本。修改本文件时,需同步更新 agent。

## 概述

`char/世界书/变量/变量更新规则.md` 是发给 LLM 的提示词,告诉 LLM 应该在什么情况下、按什么规则更新各 MVU 变量。

它的字段结构与 `char/脚本/Zod.ts` 中的 zod schema **必须保持一致**。

## 文件骨架

```yaml
---
变量更新规则:
  字段名 1:
    type: 字段类型(可选)
    其他元数据(可选): ...
    check:
      - 检查规则 1
      - 检查规则 2
  字段名 2:
    ...
```

## 字段合并规则

### 1. 同类型字段合并

`z.object({...})` 与 `z.record(z.enum([...]), ...)` 的键固定且类型相同时,可用 `${通称}` 占位符合并。

例如 zod 中:

```ts
能力面板: z.record(z.enum(['力量', '敏捷', '体质', '感知', '意志', '魅力']), z.coerce.number())
```

可以合并写为:

```yaml
能力面板.${六维}:
  type: number
  range: 0~100
  check:
    - 训练、战斗、重伤等显著事件才调整
```

### 2. 动态键字段

`z.record(z.string().describe('xx'), ...)` 的键不固定,在 `type` 中用 TypeScript 索引签名描述:

```yaml
物品栏:
  type: |-
    {
      [物品名: string]: {
        描述: string;
        数量: number;
      }
    }
  check:
    - 拾取、丢弃、消耗物品时更新
```

### 3. 嵌套字段

同一对象的字段嵌套在该对象 mapping 下,不要平铺:

```yaml
# ✅ 推荐
某角色:
  能力面板.${六维}:
    type: number
    check: [...]
  装备栏.${部位}:
    type: |-
      {
        装备: string;
        评价: string;
      }
    check: [...]

# ❌ 不推荐(平铺)
某角色.能力面板.${六维}:
  ...
某角色.装备栏.${部位}:
  ...
```

## 字段元数据约定

| 字段 | 用途 | 是否必需 |
|------|------|---------|
| `type` | 类型标注。string 类型时**省略**;其他类型为 number/boolean/或 TS 类型定义 | 非 string 时必需 |
| `range` | 数值范围,如 `0~100` | 可选 |
| `format` | 格式模板,如 `${YYYY/MM/DD}-${HH:MM}` | 可选 |
| `category` | 分级映射,如 `20~40: 普通人` | 可选 |
| `value` | 字段含义说明(string 类型常用) | 可选 |
| `check` | 更新规则列表(必需,核心字段) | **必需** |

## check 规则编写

- 简练,每条 1 句
- 描述"什么情况下应该更新""单次变化幅度""特殊约束"
- 引用 `{{user}}` 而非"主角""用户"(参见 user-replacement.md)
- 可以引用其他变量,描述联动逻辑

```yaml
check:
  - 单次变化不超过 ±10,除非剧情有明确强化/削弱
  - 仅当 {{user}} 直接接触相关事件时更新
  - 与 `某角色.信任度` 联动,后者大于 50 时本字段获得 +5 加成
```

## 不需要规则的字段

- 字段名以 `_` 开头(只读字段),**不**列出更新规则
- 字段名已经自解释、且无特殊更新限制时,**可不**列出。例如 `当前时间` 字段名已说明用途,无需另外解释

## string 类型字段简写

string 类型时省略 `type`:

```yaml
当前地点:
  check:
    - 每次场景跳转时更新
    - 描述应包含具体方位
```

## 完整示例

```yaml
---
变量更新规则:
  世界:
    当前时间:
      format: ${xx 历}-${YYYY/MM/DD}-${HH:MM}
      check:
        - 每次事件推进、休息或旅行后更新
        - 若场景跳转跨度较大,应说明跳跃原因

  主角:
    物品栏:
      type: |-
        {
          [物品名: string]: {
            描述: string;
            数量: number;
          }
        }
      check:
        - 拾取、丢弃、消耗时更新
        - 数量为 0 或负数时,该物品会被自动从物品栏移除

  某角色:
    能力面板.${六维}:
      type: number
      range: 0~100
      category:
        0~20: 病弱
        20~40: 普通人
        40~70: 训练有素
        70~100: 超凡
      check:
        - 训练、战斗、重伤、奖励等显著事件才调整
        - 单次变化不超过 ±10
    装备栏.${部位}:
      type: |-
        {
          装备: string;
          评价: string;
        }
      check:
        - 穿戴、损毁、替换时更新

  ${对象}.主角评价:
    value: 该对象对 {{user}} 的即时感受
    check:
      - 在对应变量值变化或事件触发后更新
      - 主角评价不会被 {{user}} 看到,不出现在剧情对话中
```

## 任务列表 / 事件类的写法

```yaml
任务列表:
  type: |-
    {
      [任务名: string]: {
        类型: '主线' | '支线' | '每日' | '临危受命';
        说明: string;     # 任务背景
        目标: string;     # 明确可执行的步骤
        奖励: string;
        惩罚: string;     # 失败的负面效果
      }
    }
  check:
    - 避免一次性添加超过 3 个主线任务
    - 完成后转入历史并清出列表
```
