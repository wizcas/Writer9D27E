# 世界书条目通用 Markdown + XML + YAML 格式规则

> 本文件为人类可读的规则参考手册。各 Card* agent 提示词中应内嵌本规则的副本。修改本文件时,需同步更新所有引用此规则的 agent。

## 文件类型与扩展名

每个世界书条目对应 `char/世界书/` 下的一个 `.md` 文件,职责单一,内容聚焦。

**3 个例外保留 `.yaml` / `.ts` 扩展名,且禁止 markdown 包装:**
- `char/index.yaml`(tavern_sync 同步入口)
- `char/世界书/变量/[initvar]初始化变量勿开.yaml`(MVU 框架读取的初始变量值)
- `char/脚本/Zod.ts`(可执行 TypeScript schema)

## 顶层格式(适用于 `.md` 类型条目)

文件以 Markdown 标题开头,紧接一个 XML 大块标签包裹实际数据,YAML 数据直接写在 XML 标签内(不套 ` ```yaml ``` ` 代码块):

````markdown
# <主题中文名,如 "世界设定">

<某大块>
...YAML 字段...
</某大块>
````

两层结构:
- **Markdown 标题**:让编辑器识别为 markdown 文档,可折叠
- **XML 标签**:让 LLM 高度关注此块内容(尤其 Claude 系列对 XML 标签敏感),YAML 数据直接在标签内

> **为何不套 ` ```yaml ``` `?** LLM 会把代码块识别为"示例代码",从而降低对内容的关注度。直接在 XML 内写 YAML 会让 LLM 把内容当作指令/数据来严肃对待。

## XML 标签命名

- 用英文小写 + 下划线,语义清晰。如 `<world_setting>` `<factions>` `<npc>` `<narrative_style>` `<user_profile>` `<drama_core>` `<mvu_update_rules>`
- 一个文件原则上只有一个顶层 XML 块
- 顶层块内可嵌套子块(嵌套时仍用 XML),但本项目偏好把所有结构信息放进 YAML 字段层级里,XML 只做"信息块的边界标记"

## YAML 字段规范

- 键名用中文,简短清晰
- 值优先用结构化数据(对象、列表),避免大段散文
- 长文本用 `|-` 字符串块保留换行
- 列表项 ≤ 5 个时用 `[a, b, c]` 行内格式;> 5 个时用多行 `- a` 格式
- 嵌套深度尽量 ≤ 4 层

## 内容编排原则

1. **单一职责**:一个文件只讲一件事。世界设定文件不写具体 NPC 详情,NPC 文件不写世界规则
2. **结构化优先**:能用字段表达的不用散文。`年龄: 30` 而非 `他三十岁了`
3. **维度对齐**:同类信息用相同字段名。所有 NPC 都有 `identity / appearance / personality / relationship` 等维度
4. **避免冗余**:不要在多个文件里重复同一信息。例如 NPC 与 `{{user}}` 的关系写在 NPC 文件里,不要在主角设定文件里重复

## 字符串值规范

- 短描述:不加引号
- 含特殊字符(如 `:`、`#`、`{`、`}`、`[`、`]`)或前后空格的字符串:用单引号或双引号
- 多行文本:用 `|-`(保留换行,删除末尾空行)或 `>-`(折叠为单行)
- 引用 `{{user}}` 等酒馆宏的字符串:推荐加引号避免解析歧义,如 `'{{user}} 走进房间'`

## 字段命名建议(参考性,非强制)

- 身份维度:`identity` / `身份` / `基础信息`
- 外观维度:`appearance` / `外观` / `physical trait`
- 性格维度:`personality` / `性格` / `key trait`
- 关系维度:`relationship` / `关系` / `social connection`
- 行为倾向:`behavioral tendency` / `行为模式`
- 语言风格:`language style` / `说话方式`

## 完整示例骨架

````markdown
# 某主题

<某大块 name="某子项标识">
子块_1:
  字段名 1: 值
  字段名 2:
    - 列表项 1
    - 列表项 2
  字段名 3:
    子字段 a: 值
    子字段 b: 值

子块_2:
  字段名: |-
    多行文本第一行
    多行文本第二行
</某大块>
````

## 关于 markdown 之外的注释段落

可选:在 XML 标签外可加 Markdown 段落作为编辑者注释(给阅读源文件的人看)。但请记住,**整个 `.md` 文件全文都会被发给 LLM**(tavern_sync 不会过滤 XML 外的内容),所以注释段落也会被 LLM 读到。如不希望发送,改用 HTML 注释 `<!-- ... -->` 或 yaml 代码块外的中文段落。

## 反模式(禁止)

- ❌ 整段连续散文(没有任何字段结构)
- ❌ 一个文件混杂多个主题(如同时写世界观和具体 NPC)
- ❌ XML 标签未闭合
- ❌ YAML 缩进混乱(空格数不一致)
- ❌ 在文件名或文件正文里出现 `{{user}}` 字面量以外形式的"主角""用户"指代(参见 user-replacement.md)
- ❌ 错误的扩展名(应为 `.md`,除 3 个例外)
- ❌ 在 XML 标签内用 ` ```yaml ``` ` 代码块包裹数据(会让 LLM 把数据当作示例代码)
