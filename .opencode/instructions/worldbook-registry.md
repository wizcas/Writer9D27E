# 世界书条目注册规则(写入文件后同步更新 index.yaml)

> 本文件为人类可读的规则参考手册,对应 AGENTS.md 强制规则 9。
> Card* agent 写入产物文件后必须按本文件规则更新 `char/index.yaml`。

## 职责边界

| 角色 | 职责 |
|------|------|
| **CardIndex** | 全量重建 `char/index.yaml`,拥有对该文件的完整控制权 |
| **其他 Card* agent** | 增量追加:仅在 `条目` 数组末尾追加自身产物对应的新条目,不得修改已有条目 |

## 操作流程

1. 写入产物文件(`.md` / `.yaml` / `.ts`)后,立即用 Read 工具读取 `char/index.yaml`
2. 若 `char/index.yaml` 不存在 → 停止,告知用户"请先运行 CardIndex 生成 index.yaml"
3. 在 `条目` 数组(含嵌套 `文件夹` 内的 `条目`)中查找是否已有 **`名称`** 与待注册条目相同的项:
   - **已存在** → 跳过,不做任何修改
   - **不存在** → 在适当位置追加新条目(见各类型默认值)
4. 用 Write 工具将更新后的内容写回 `char/index.yaml`

## 路径写法规则

- 路径**不带**文件扩展名(`.md` / `.yaml`)
- 路径相对于 `char/` 根目录
- 示例:
  - `char/世界书/世界设定.md` → `文件: 世界书/世界设定`
  - `char/世界书/NPC/洛洛.md` → `文件: 世界书/NPC/洛洛`
  - `char/世界书/变量/变量更新规则.md` → `文件: 世界书/变量/变量更新规则`
  - `char/世界书/变量/[initvar]初始化变量勿开.yaml` → `文件: 世界书/变量/[initvar]初始化变量勿开`

## 各文件类型的条目默认值

### 世界设定 / 势力设定 / 主角设定

```yaml
- 名称: '[世界书]世界设定'          # 方括号内填写分类,后跟文件名
  启用: true
  激活策略:
    类型: 蓝灯
  插入位置:
    类型: 指定深度
    角色: 系统
    深度: 4
    顺序: 100
  激活概率: 100
  递归:
    不可被其他条目激活: false
    不可激活其他条目: false
  文件: 世界书/世界设定
```

势力设定与主角设定格式相同,仅 `名称` 与 `文件` 字段不同:

```yaml
- 名称: '[世界书]势力设定'
  # ...同上...
  文件: 世界书/势力设定

- 名称: '[世界书]主角设定'
  # ...同上...
  文件: 世界书/主角设定
```

### NPC

NPC 条目放入 `NPC` 文件夹分组。若 `条目` 数组中尚无 `文件夹: NPC` 分组,则在追加时一并创建该分组:

```yaml
- 文件夹: NPC
  条目:
    - 名称: '[NPC]洛洛'             # 方括号内填 NPC,后跟 NPC 名
      启用: true
      激活策略:
        类型: 蓝灯
      插入位置:
        类型: 角色定义之后
        顺序: 100
      激活概率: 100
      递归:
        不可被其他条目激活: false
        不可激活其他条目: false
      文件: 世界书/NPC/洛洛
```

若 `NPC` 文件夹已存在,则仅在其 `条目` 数组末尾追加新 NPC 条目。

### 第一条消息(例外)

CardFirstMessage 写入的文件**不属于** `条目` 数组,而是顶层 `第一条消息` 字段。注册方式:

1. 读取 `char/index.yaml` 中的 `第一条消息` 数组
2. 检查是否已含 `{ 文件: 第一条消息/0 }` 项
3. 若不存在则追加:

```yaml
第一条消息:
  - 文件: 第一条消息/0
```

## 去重判断逻辑

- 去重字段:**`名称`**
- 匹配范围:递归扫描所有顶层 `条目` 及嵌套 `文件夹` 内的 `条目`
- 匹配方式:字符串完全相等(区分大小写)
- 若找到同名条目 → 跳过,不追加,不修改原条目

## 不适用本规则的情况

- **CardIndex**:本就负责全量重建 index.yaml,不受强制规则 9 约束
- **CardMvuSchema**:输出 `char/脚本/Zod.ts`,该文件不是世界书条目,无需注册到 `条目` 数组
