# index.yaml 固定格式规则(tavern_sync)

> 本文件为人类可读的规则参考手册。CardIndex agent 提示词中应内嵌本规则的副本。修改本文件时,需同步更新 agent。

## 概述

`char/index.yaml` 由 `tavern_sync` 工具读取,用于将文件系统的角色卡内容同步到酒馆 (SillyTavern) 应用。其格式**完全固定**,禁止添加自定义字段。

## 文件首行

必须有 schema 引用注释:

```yaml
# yaml-language-server: $schema=https://testingcf.jsdelivr.net/gh/StageDog/tavern_sync/dist/schema/character.zh.json
```

## 顶层必需字段

| 字段 | 类型 | 说明 |
|------|------|------|
| 头像 | string \| null | 头像文件名(不含扩展名),无则 null |
| 版本 | string | 角色卡版本号,如 '1.0.0' |
| 作者 | string | 作者署名 |
| 备注 | string | 给玩家看的简短备注 |
| 第一条消息 | array | 第一条消息文件列表,每项 `{ 文件: 路径 }` |
| 角色描述 | string | 通常留空 '',因为内容在世界书里 |
| 锚点 | array(可选) | YAML anchor 复用配置块 |
| 世界书名称 | string | 世界书在酒馆中的名称,可填 '与角色卡名称相同' |
| 条目 | array | 世界书条目列表 |
| 扩展字段 | object(可选) | 正则、酒馆助手脚本库等 |

## 第一条消息字段

```yaml
第一条消息:
  - 文件: 第一条消息/0    # 不带 .md 扩展名,相对 char/ 根
```

可有多个文件,玩家可在酒馆中切换。

## 锚点(YAML anchor)

用于复用重复的条目配置块:

```yaml
锚点:
  - &分隔符
    启用: false
    激活策略:
      类型: 向量化
    插入位置:
      类型: 角色定义之前
      顺序: 14720
    激活概率: 100
    内容: ''
  - &立即事件
    启用: true
    插入位置:
      类型: 指定深度
      角色: 系统
      深度: 0
      顺序: 14720
    激活概率: 100
    特殊效果:
      黏性: 5
      冷却: 9999
    递归:
      不可被其他条目激活: true
      不可激活其他条目: true
```

引用方式:

```yaml
- 名称: ===世界设定===
  <<: *分隔符

- 名称: 某事件
  <<: *立即事件
  激活策略:
    类型: 绿灯
    关键字: ['【【某事件】】']
  文件: 世界书/某事件
```

## 条目必需字段

每个条目必需以下字段:

```yaml
- 名称: 条目名(显示在酒馆 UI)
  启用: true                  # boolean
  激活策略:
    类型: 蓝灯|绿灯|向量化
    # 若为绿灯,需:
    # 关键字: [关键词 1, 关键词 2]
  插入位置:
    类型: 角色定义之前|角色定义之后|指定深度
    顺序: 14720               # 数字,决定条目排序
    # 若为指定深度,需:
    # 角色: 系统|用户|AI
    # 深度: 0
  激活概率: 100              # 0-100
  递归:
    不可被其他条目激活: true
    不可激活其他条目: true
  文件: 世界书/某文件         # 不带 .md 扩展名,相对 char/ 根
```

## 文件夹分组

可用 `文件夹` + `条目` 嵌套来分组:

```yaml
- 文件夹: 变量
  条目:
    - 名称: ===变量开始===
      <<: *分隔符
    - 名称: '[initvar]初始化变量勿开'
      启用: false
      ...
      文件: 世界书/变量/initvar
    - 名称: ===变量结束===
      <<: *分隔符
```

## MVU 启用时必含的脚本库条目

```yaml
扩展字段:
  酒馆助手:
    脚本库:
      - 名称: mvu
        id: 22d70d1f-d0be-4061-911e-d0c57ab55053
        启用: true
        类型: 脚本
        内容: |-
          import 'https://testingcf.jsdelivr.net/gh/MagicalAstrogy/MagVarUpdate/artifact/bundle.js';

      - 名称: 变量结构
        id: <生成新的 UUID>
        启用: true
        类型: 脚本
        内容: |-
          import '<最终发布后的脚本 URL,留空或占位即可>';
```

注意:`变量结构` 脚本的 `内容` 字段中的 import URL 需要在打包发布后填入实际地址,生成时可填入占位符或留待用户后续修改。

## 顺序数值的含义

`顺序` 字段决定同一 `插入位置类型` 下各条目的排列顺序:

- **数值越小 → 插入位置越靠前(远离最新消息)→ LLM 较早读到,影响较弱**
- **数值越大 → 插入位置越靠后(靠近最新消息)→ LLM 较晚读到,影响更大**

因此,希望 LLM 重点关注、优先遵守的条目(如叙事风格、变量更新规则),应使用**较大**的顺序数值;背景设定等辅助条目可使用较小数值。

调整策略:在不改变其他选项的前提下,单独调大或调小 `顺序` 值,可让条目整体上移或下移,进而影响 LLM 对该条目的重视程度。也可以结合 `插入位置` 的其他选项(如插入位置类型、深度等)综合调整条目最终落点。

## 插入位置 / 顺序的建议值

- 通用世界设定类(世界、势力、主角设定):`角色定义之后`,顺序 14720+
- 叙事风格、文风:`指定深度 / 系统 / 0`,顺序 14720
- 立即事件类(关键字触发):`指定深度 / 系统 / 0`,顺序 14720,加 `特殊效果` 与 `递归: 不可激活其他条目`
- MVU 系列条目:见上一节

## 禁止事项

- 禁止在 index.yaml 中写非 tavern_sync schema 定义的字段
- 禁止在 index.yaml 中写实际世界书内容(内容应在 `世界书/*.md` 中)
- 禁止文件路径含 `{{user}}` 字面量(用 `user`)
