# {{user}} 替换强制规则

> 本文件为人类可读的规则参考手册。各 Card* agent 提示词中应内嵌本规则的副本。修改本文件时,需同步更新所有引用此规则的 agent。

## 核心原则

在世界书条目、第一条消息、initvar 等所有"会被发送给 LLM"的正文内容中,**禁止**直接出现以下指代用户的字样,必须替换为 `{{user}}`:

| 禁止词汇 | 替换为 |
|---------|--------|
| 主角 | `{{user}}` |
| 男主 / 女主 | `{{user}}` |
| 男主角 / 女主角 | `{{user}}` |
| 用户 | `{{user}}` |
| 玩家 | `{{user}}` |
| 你(指代用户时) | `{{user}}` |
| 主人公 | `{{user}}` |
| pc / PC / Player Character | `{{user}}` |

## 例外:允许保留原词的场景

以下场景中即使出现"主角""用户"等字样也允许保留,因为它们不是发给 LLM 的正文,或不会引起歧义:

1. **文件路径**:`角色卡/主角配置.md` 这类路径名(但本项目要求路径用 `char/` 且文件名不含 `{{user}}`)
2. **文件名**:文件名中需要表达"用户"概念时使用 `user`(无双花括号)。例如 `user设定.md`
3. **YAML 字段键名**:作为结构化数据键名时可保留。例如 `主角评价: ...` 作为字段名是可以的,但其字符串值里必须用 `{{user}}`
4. **XML 标签名**:`<user_profile>` 这类标签名可保留,但标签内部正文必须替换
5. **章节标题 / Markdown 标题**:`## 主角设定` 这类标题可保留
6. **本规则文件本身**:讨论规则时不算正文

## 边界判定示例

```yaml
# ✅ 正确写法
<user_profile name="user">
  identity:
    年龄: 30
    职业: 自由职业者
  relationship:
    与女主的关系: "{{user}} 与她从小一起长大"   # 字段名"与女主的关系"是 key,可保留;值里替换
  主角评价:                                     # 字段名作为 key,可保留
    描述: "{{user}} 内心柔软,但表面随和"        # 值里替换
</user_profile>

# ❌ 错误写法
<user_profile>
  description: "主角是一个三十岁的男人"        # 正文里有"主角",必须改为 {{user}}
  关系网:
    女主: "用户与她从小一起长大"               # 正文里有"用户",必须改为 {{user}}
</user_profile>
```

## 自检流程

每次写完一个文件后,执行以下检查:

1. 用 grep 扫描禁词:`grep -E "(主角|男主|女主|用户|玩家|主人公)" <文件路径>`
2. 对扫描出的每一处,判断它是 [字段名/标签名/路径/标题] 还是 [正文]
3. 若属于正文,改为 `{{user}}`
4. 重新扫描,直至所有正文中的禁词均被替换
