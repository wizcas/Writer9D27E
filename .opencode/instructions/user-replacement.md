# {{user}} 替换强制规则

> 本文件为人类可读的规则参考手册。各 Card* agent 提示词中应内嵌本规则的副本。修改本文件时,需同步更新所有引用此规则的 agent。

## 核心原则

在世界书条目、第一条消息、initvar 等所有"会被发送给 LLM"的正文内容中,**禁止**直接出现以下指代用户的字样,必须替换为 `{{user}}`:

| 禁止词汇 | 替换为 |
|---------|--------|
| 主角 | `{{user}}` |
| 男主 / 女主 | `{{user}}` |
| 男主角 / 女主角 | `{{user}}` |
| 用户 | `{{user}}` |
| 玩家 | `{{user}}` |
| 你(指代用户时) | `{{user}}` |
| 主人公 | `{{user}}` |
| pc / PC / Player Character | `{{user}}` |

## 例外:禁止使用 `{{user}}` 必须使用 `主角` 的场景

在以下特定的结构化标记或配置中，**绝对禁止**使用 `{{user}}` 以及英文 `user`，**必须**使用名词 `主角` 代替：

1. **所有文件名**(例如:`char/世界书/主角设定.md`，不能用 `user设定.md` 或 `{{user}}设定.md`)
2. **`index.yaml`(即 tavern_sync 配置)中的键(Key)和条目标题(Title / name)**(例如: `name: "[世界书]主角设定"`)
3. **MVU、Zod 及变量系统中的键名(Key)**(例如: `主角: z.object(...)` 代替 `user: z.object(...)`，`主角.物品栏` 代替 `user.物品栏`)
4. **XML 标签名**:`<protagonist_profile>` 这类标签名使用英文

**除上述明确指出的例外场景外，世界书的内部文本、YAML 键名(如 `与 {{user}} 关系`)、Markdown 标题(如 `# {{user}} 设定`)、变量的 value/describe 等，均必须使用 `{{user}}`，禁止使用 `主角`、`男主` 等。**

## 边界判定示例

```yaml
# ✅ 正确写法
<protagonist_profile>
  identity:
    年龄: 30
    职业: 自由职业者
  relationship:
    与女主的关系: "{{user}} 与她从小一起长大"
  {{user}}评价:                                  # 世界书中普通的 YAML 键名必须用 {{user}}
    描述: "{{user}} 内心柔软,但表面随和"
</protagonist_profile>

# ❌ 错误写法
<user_profile>
  description: "主角是一个三十岁的男人"        # 正文里有"主角",必须改为 {{user}}
  与 user 关系: "中立"                         # 键和值中的 user 都必须改为 {{user}}
  主角评价:                                    # 世界书 YAML 键名错误使用了"主角",应改为 {{user}}评价
    描述: "主角内心柔软"                       # 错误使用了"主角",必须改为 {{user}}
</user_profile>
```

```typescript
// ✅ 正确写法 (Zod.ts 中变量键名)
主角: z.object({                                // MVU 变量键名必须用"主角"，不能用 user 或 {{user}}
  物品栏: z.record(z.string(), z.object({
    描述: z.string().describe("{{user}} 携带的物品") // 变量的描述/值里必须用 {{user}}
  }))
})
```

## 自检流程

每次写完一个文件后,执行以下检查:

1. 用 grep 扫描禁词:`grep -E "(主角|男主|女主|用户|玩家|主人公)" <文件路径>`
2. 对扫描出的每一处,判断它是 [字段名/标签名/路径/标题] 还是 [正文]
3. 若属于正文,改为 `{{user}}`
4. 重新扫描,直至所有正文中的禁词均被替换
