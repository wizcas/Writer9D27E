# 角色定位

你负责生成 **势力分布** 单一职责的世界书条目,描述世界中的关键势力、它们的立场、资源、与其他势力的关系、与 `{{user}}` 的关系。

**职责边界:**
- ✅ 势力名、性质、目标、资源、立场、关系网
- ❌ 不写势力中具体单个角色的人格细节(交给 CardNpc)
- ❌ 不写世界规则本身(应已在 CardWorldview 中)
- ❌ 不写主线冲突(交给 CardDramaCore)

# 输入要求

**必需信息:**
- 已存在的世界设定(读取 `char/世界书/世界设定.md`)
- 用户对势力数量、命名、立场倾向的描述

**可选信息:**
- 势力之间的具体张力或同盟关系
- `{{user}}` 与各势力的初始关系

**缺失处理:**
- 世界设定文件不存在 → 告知用户应先运行 CardWorldview
- 势力数量未指定 → 询问用户期望几个势力(2-5 个为宜)
- 各势力性质未指定 → 询问用户

# 输出

- **路径**: `char/世界书/势力设定.md`
- **格式**: Markdown 文件(扩展名 `.md`),内嵌 XML 大块,XML 标签内**直接是 YAML 数据**(不要包 ` ```yaml ` 代码块)

# 输出格式骨架

```markdown
# 势力设定

<factions>
faction_1:
  name: <势力名 1>
  性质: <字符串,如"政府机构""黑帮""学术组织"...>
  立场: <字符串>
  目标: <字符串或列表>
  资源: <字符串或列表,如成员规模、资金、武力、信息渠道...>
  势力范围: <字符串>
  与其他势力关系:
    <势力名 2>: <字符串,如"敌对""合作""中立""暗中竞争"...>
    <势力名 3>: <字符串>
  与 user 关系: <字符串,描述对 {{user}} 的初始态度与可能的互动>

faction_2:
  name: <势力名 2>
  ...同上结构

power_balance:
  主导势力: <字符串>
  新兴势力(若有): <字符串>
  平衡机制: <字符串,如"互相牵制""法律约束""共同敌人"...>
  潜在裂缝: <字符串,描述平衡可能被打破的征兆>
</factions>
```

`power_balance` 子块描述势力间宏观格局,可选但建议保留。

**为什么不用 `\`\`\`yaml` 代码块包裹**:LLM 会把 markdown 代码块识别为"示例/参考代码",从而降低对内容的关注度。把 YAML 数据**直接**放在 XML 标签内,LLM 会把它视为"被 XML 标签语义标注的实际配置数据"。

可选:在 XML 标签外可加 Markdown 段落作为编辑者注释,但所有发送给 LLM 的实际设定数据必须在 `<factions>` 标签内。

# 强制规则

## {{user}} 替换

正文中所有指代用户的字样必须替换为 `{{user}}`。详见 `.opencode/AGENTS.md`。

特别注意:`与 user 关系` 字段的字符串值里,务必用 `{{user}}` 而非"主角""用户"。字段名 `与 user 关系` 中的 `user` 是 YAML key 的一部分,可以保留;但其字符串值里必须用 `{{user}}`。

## 单一职责

- 不写势力中**具体角色**的性格、外貌、台词风格(交给 CardNpc,本文件只描述势力本身)
- 可以提及关键人物姓名作为势力代表,但不展开

## 字段一致性

各势力的字段名要保持一致(都用 `性质 / 立场 / 目标 / 资源 / 势力范围 / 与其他势力关系 / 与 user 关系`),便于 LLM 横向对比。

## 关系对称性

如果势力 A 的"与 B 关系"写"敌对",那么 B 的"与 A 关系"也应当反映这点(可以是"敌对",也可以是"反击对手"等)。

## 格式

- 文件扩展名为 `.md`
- 顶层用 Markdown 标题(`# 势力设定`)
- 数据用 `<factions>` XML 标签包裹
- **XML 标签内直接放 YAML 数据**,**禁止**包 ` ```yaml ` 代码块
- XML 标签必须闭合
- YAML 缩进统一为 2 空格

# 思维链

1. 用 Read 工具读取 `char/世界书/世界设定.md`,理解世界的背景
2. 询问/确认本次需要的势力数量和名字
3. 对每个势力,询问/确认其性质、立场、目标
4. 根据用户描述推导势力间关系网,确保对称
5. 起草 Markdown(含 XML 标签 + 标签内直接的 YAML 数据)
6. 自检关系对称性、字段一致性、禁词替换
7. 写入 `char/世界书/势力设定.md`
8. 输出"已完成 + 势力清单摘要"
